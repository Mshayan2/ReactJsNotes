Introduction to React.js

React is an open-source JavaScript library developed by Meta (Facebook) to construct dynamic, component-driven user interfaces.

  • Created: 2011 by Jordan Walke. Open-sourced in 2013.
  • Purpose: Solves the complexity of updating stateful DOM trees in large-scale applications.
  • Application style: Powers Single-Page Applications (SPAs).

🧩 Component-Based

Divide complex layouts into isolated, reusable blocks of code that handle their own states.

⚡ Declarative Paradigm

Describe how your UI should look for a given state. React handles the heavy lifting of updating the real DOM.

🌐 Virtual DOM

Keeps lightweight copies of the DOM in memory to calculate updates before rendering them. Unmatched rendering speed!

Installing React (CRA vs Vite)

Vite is a modern frontend build tool that is incredibly fast due to ESM-based hot module replacement.

# 1. Initialize project
npm create vite@latest my-app -- --template react

# 2. Enter folder
cd my-app

# 3. Install packages
npm install

# 4. Start local development server
npm run dev

🌐 Local port: http://localhost:5173

Why choose Vite?

  • Instant server starts.
  • Extremely fast build and hot reloading (HMR).
  • Very small bundle sizes.
  • Leverages native ES modules directly.

Create React App (CRA) is the traditional configuration by Meta. It utilizes Webpack under the hood.

# 1. Initialize and build workspace
npx create-react-app my-app

# 2. Enter folder
cd my-app

# 3. Start local development server
npm start

🌐 Local port: http://localhost:3000

CRA Characteristics

  • Pre-configured Webpack and Babel setups.
  • Larger node_modules size.
  • Slower build times for large applications.
  • Mostly deprecated by Meta in favor of Vite, Next.js, or Remix.

React Fragment

React requires every component to return a single parent container. Standard containers like `<div>` add redundant layers to the web browser DOM tree.

Short syntax representation:

function App() {
  return (
    <>
      <h1>Welcome</h1>
      <p>This is my app</p>
    </>
  );
}

Why use Fragments?

  • Cleaner DOM: Eliminates wrapper divs.
  • Correct HTML structure: Essential when rendering children inside CSS Flex/Grid containers or tables where wrapping divs break semantics.
  • Long Syntax Support: Use `React.Fragment` if you need to attach `key` properties to list items:
<React.Fragment key={item.id}>
  <dt>{item.term}</dt>
  <dd>{item.desc}</dd>
</React.Fragment>

Working with Images

1. Asset Import (src folder)

Best for localized component assets. Webpack/Vite checks, optimises, and compiles these files at build time.

import myPic from './assets/photo.jpg';

function Profile() {
  return <img src={myPic} alt="My Photo" />;
}

2. Public Folder Reference

Best for dynamic, universal site assets. Assets are stored intact and requested direct from root url.

// Looks inside the 'public' folder directly
function Profile() {
  return <img src="/photo.jpg" alt="My Photo" />;
}

💡 Architectural Rules on Asset Locations:

Use the import method by default for images tied to components (profile images, cards). Use the public folder for assets that must maintain fixed, static URLs or are referenced globally (manifest, icons, favicon, static JSON).

CSS Modules & className

Normal CSS styles are global and create selector conflicts. CSS Modules scope your styles locally to preventing leakage.

App.module.css

.title {
  color: var(--primary);
  font-size: 24px;
}

App.jsx usage

import styles from './App.module.css';

function App() {
  return <h1 className={styles.title}>Hello CSS Modules</h1>;
}

🏷 Why className?

JSX compiled output resolves to pure JavaScript. In JavaScript, class is a reserved keyword for ES6 classes. Thus, React JSX maps standard HTML classes to className.

Global CSS Leakage

Styles collide when selectors match across elements, styling page-wide.

React Hooks Introduction

React Hooks are special functions that allow you to "hook into" React's local state, components lifecycle, context features directly inside functional components.

💡 Paradigm Shift

Prior to React 16.8, only Class components could hold local state. Hooks enabled functional components to achieve complete feature parity with classes without using heavy boilerplate, construct code bases, or handling the confusing this keyword.

⚠️ Crucial Rules of Hooks

  • Only call Hooks at the top level of your component. Don't call them inside loops, condition blocks, or nested blocks.
  • Only call Hooks from React functional components or custom Hooks. Never call them from normal JS functions.

Core Hooks Overview:

useState: Local state management
useEffect: Run side-effects
useContext: Access global Context
useRef: Select DOM nodes / non-re-rendering states
🔹

The useState Hook

Allows components to preserve local variables that, when updated, automatically trigger a template re-render in the DOM viewport.

Syntax

const [state, setState] = useState(initialValue);

🏏 Cricket Scoreboard Analogy

Imagine a live match scoreboard:

  • state (count): The current runs score shown on the screen.
  • setState (setCount): The official scorer who clicks/types updates to push a new value to the board.
import React, { useState } from "react";

function Counter() {
  const [count, setCount] = useState(0);

  return (
    <div>
      <h2>Count: {count}</h2>
      <button onClick={() => setCount(count + 1)}>
        Increase
      </button>
    </div>
  );
}

React Props: Introduction

Props (properties) are parameters passed into components, operating like HTML attributes or arguments in a standard JavaScript function.

📺 TV Remote Analogy

A TV component renders standard displays. The buttons pressed (Volume Up, Channel 5) act like props. They do not rebuild a new TV; they adjust parameter states inside the component layout.

⚠️ Absolute Rules:

Props are strictly read-only (immutable). Child components cannot mutate props passed from their parents.

Simple Example:

function Welcome(props) {
  return <h2>Hello, {props.name}!</h2>;
}

function App() {
  return (
    <div>
      <Welcome name="Shayan" />
      <Welcome name="Ali" />
    </div>
  );
}
🔹

Types of Props

  • (a) Strings: passed directly:
    <Greeting user="Shayan" />
  • (b) Numbers: wrapped in curly braces:
    <Age value={30} />
  • (c) Booleans: evaluates logically:
    <Status online={true} />
  • (d) Objects: passed as complex bindings:
    <Student data={{ name: "Ali", age: 22 }} />
  • (e) Arrays: mapping datasets:
    <Skills list={["React", "Node"]} />
  • (f) Functions (Callbacks): pass triggers to children:
    <Button onClick={handleAlert} />
  • (g) Defaults: fallback values in templates:
    Message.defaultProps = { text: "Hi" }
  • (h) Children: items nested inside:
    <Card><p>Inside Container</p></Card>

props.children deep-dive:

Inside the child component definition, props.children outputs whatever markup was nested inside the opening and closing tag of that component inside the parent.

State Lifting (Child ⇄ Parent)

React has unidirectional data flow (Parent to Child). When sibling components must communicate, we lift state to their nearest common ancestor.

How it operates:

  • The parent houses the shared state variable.
  • The parent passes down the state update function to Child A as a prop callback.
  • Child A triggers the callback with updated arguments.
  • Parent updates state, causing Parent and Child B to re-render.

Event Handling in React

React events use camelCase syntax (e.g. `onClick`, `onChange`) instead of lowercase HTML defaults.

Syntax comparison:

// HTML
<button onclick="handleClick()">Click</button>

// React (JSX)
<button onClick={handleClick}>Click</button>

⚠️ Avoid Direct Invocation:
onClick={handleClick()} runs instantly at load time. Always pass the function reference: onClick={handleClick}.

The useEffect Hook

Used to perform side-effects (data fetching, DOM updates, event listeners, intervals) inside functional components.

useEffect(() => {
  // 1. Side Effect code here
  fetchData();

  // 2. Optional clean-up function
  return () => cleanup();
}, [dependencies]);

Dependency Array configurations:

  • [ ]: Runs once at mount time.
  • [stateVar]: Runs at mount AND whenever stateVar shifts value.
  • None: Runs after every single render cycle.

React Hook Form

Simplifies form handling and field validation in React, avoiding redundant component re-renders that happen when using controlled state inputs.

import { useForm } from 'react-hook-form';

function Form() {
  const { register, handleSubmit, formState: { errors } } = useForm();
  const onSubmit = data => console.log(data);

  return (
    <form onSubmit={handleSubmit(onSubmit)}>
      <input {...register("name", { required: true })} />
      {errors.name && <span>Required</span>}
      <button>Submit</button>
    </form>
  );
}

useRef, useMemo & useCallback

useRef

Persists mutable values across renders WITHOUT causing component re-render triggers. Also directly references actual HTML DOM elements.

const inputRef = useRef();
// Focus element
inputRef.current.focus();

useMemo

Caches the result of an expensive computation, re-calculating it only when variables in the dependency array change value.

const computedVal = useMemo(() => {
  return expensiveCalc(val);
}, [val]);

useCallback

Caches the function definition itself to prevent children components from re-rendering due to changing memory address references.

const handleToggle = useCallback(() => {
  setOpen(o => !o);
}, []);

Key Takeaways & Wrap Up

React Checklist

  • Always think in declarative state structures, not procedural DOM updates.
  • Maximize components reusability. Keep files focused.
  • Never update state variables directly. Always use the hook update callbacks.
  • Keep your DOM clean by nesting layout siblings in Fragments.
